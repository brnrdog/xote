<div>
    <h2 id="what-is-xote"> {"What is Xote?"} </h2>
    <p>
      {"Xote is a UI library for ReScript built around fine-grained reactivity. It re-exports "}
      <a href="https://brnrdog.github.io/rescript-signals" target="_blank"> {"rescript-signals"} </a>
      {" for state, derived values, and effects, then adds the pieces you need to build applications: components, JSX support, routing, SSR, and hydration."}
    </p>
    <p>
      {"The design goal is simple: keep the reactive model small and explicit, then let updates flow directly to the DOM instead of re-running whole component trees."}
    </p>
    <ul>
      <li>
        {"Signals for local and shared state"}
      </li>
      <li>
        {"Computed values for derived state"}
      </li>
      <li>
        {"Effects for external side effects"}
      </li>
      <li>
        {"View primitives and JSX for UI composition"}
      </li>
      <li>
        {"A built-in router, SSR, and hydration"}
      </li>
    </ul>

    <h2 id="quick-example"> {"Start Here"} </h2>
    <p>
      {"This counter shows the core model: a signal stores state, an event updates it, and a reactive view node reads it."}
    </p>
    <h3 id="using-jsx-syntax"> {"Using JSX Syntax"} </h3>
    
```rescript
open Xote

let count = Signal.make(0)

let increment = (_evt: Dom.event) => {
  Signal.update(count, n => n + 1)
}

let app = () => {
  <div>
    <h1> {View.text("Counter")} </h1>
    <p>
      {View.signalText(() => `Count: ${Signal.get(count)->Int.toString}`)}
    </p>
    <button onClick={increment}>
      {View.text("Increment")}
    </button>
  </div>
}

View.mountById(app(), "app")
```

    <p>
      {"When "}
      <code> {"count"} </code>
      {" changes, only the reactive view node updates. The component does not need a render loop or a dependency array."}
    </p>

    <h2 id="philosophy"> {"Philosophy"} </h2>
    <p>
      {"Xote keeps the runtime surface small and explicit. State lives in signals, derived state lives in computeds, and external work lives in effects. That separation makes update paths easier to follow and easier to debug."}
    </p>
    <p>
      {"Because Xote builds on "}
      <a href="https://brnrdog.github.io/rescript-signals" target="_blank"> {"rescript-signals"} </a>
      {", the reactive core stays close to the broader signals direction in JavaScript while exposing a UI API that feels natural in ReScript."}
